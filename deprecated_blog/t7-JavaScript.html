Have a Cup of JavaScript

by Santa Maria, Steven * August 21, 2015

Recently in our Dev Bootcamp course, we jumped out of the world of Ruby and plopped into Javascript. There are definitely many similarities between Ruby and JavaScript thankfully; and as a result, making the syntatic and logical transition from the former is easier than I expected. One difference, however, that caught my attention was the relationship between Ruby hashes and JavaScript objects.

Ruby can easily employ a type of data structure called a hash. Hashes contain key-value pairs, which can be transformed. Here is an example of a Ruby hash:

grades = { "Jane Doe" => 10, "Jim Doe" => 6 }

In this example, "grades" is our hash name; "Jane Doe" and "Jim Doe" are our keys; and "10" and "6" are their values respectively.

JavaScript uses array data structures quite well, but hash data structures are another story. Hashes data structures in JavaScript are difficult to create and are rarely used.

What I noticed though was that JavaScript objects are themselves (like) hashes. Let's look an example of a JavaScript object:

var teaMaster = {
weight: 140,
hasTeaBowl: true,
hasBambooWhisk: true,
}

Looks reminiscent of our data hashes in Ruby, doesn't it? Another key difference is that we can dynamically add properties to the object e.g.

teaMaster["age"] = 74;

Doing so would add the new property "age" and its value "74" to the teaMaster object at runtime.

Learning Ruby and JavaScript have the inherent challenges of wrapping our heads around different syntax. However, it also carries the challenge of figuring out the capabilities, limitations, and logical differences between the two.
