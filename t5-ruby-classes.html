Class is in Session

by Santa Maria, Steven * August 7, 2015

Bingo. A timeless game of chance and profitable fundraisers for non-profits. But! the game can be an awesome illustration of the value of Classes in Ruby.

As with any bingo game, as a player, we need a board. Heck, I have one in my pocket right here:

board =
[[47, 44, 71, 8, 88],
[22, 69, 75, 65, 73],
[83, 85, 97, 89, 57],
[25, 31, 96, 68, 51],
[75, 70, 54, 80, 83]]

Bingo boards look awful like arrays, don't they? This will come into play later in our code.

First we'll want to declare a new class. Let's call it "BingoBoard" and we'll initialize our pocket-sized board.

class BingoBoard

def initialize(board)
@bingo_board = board
end

We can't have a bingo game without those balls that go round-and-round in the cage. Let's make a method for our BingoBoard class.

def generate_ball
bingo_array = %w(b i n g o)
@letter = bingo_array.sample
@number = rand(100)
puts "You got #{@letter}, #{@number}!"
end

Equally crucial, we need a new method to check the result of "generate_ball" against the board. How do we do that? true to the programming world, there are many ways to do it. This is what I came up with:

def check
raise ArgumentError, "You need to generate a ball
first!" if @letter == nil || @number == nil
bingo_hash = { "b"=>0, "i"=>1, "n"=>2, "g"=>3, "o"=>4 }
check_for_letter = bingo_hash[@letter]
@bingo_board.map {|outside_element|
if outside_element.kind_of?(Array)
outside_element[check_for_letter] = "X" if outside_element[check_for_letter] == @number
end }
@bingo_board.map { |outside_element| puts outside_element.inspect }
end

For the sake of convenience, we'll want a third method to display our board. Let's call it "display_board".

def display_board
@bingo_board.map { |outside_element| puts outside_element.inspect }
end
end # this ends the Class definition

It's a fair amount of code and I don't want to get into the details of it. What I do want to highlight however is how it all comes together. We have Class called BingoBoard that has all the functions required: a way to "generate_ball", to "check" the ball against the board, and a method to "display_board".

To put it another way, Classes help organize our code into more cohesive stories. The story is changed by invoking a different board!

Classes are sometimes hard to wrap our minds around at first. However, stick with them--! they're important for function and context!
